Technical Design — Berlin Leben

Architecture, Claude integration, spaced repetition, and data model

Architecture Overview

React components, TypeScript, Tailwind CSS. All UI rendering, exercise interactions, animations.

⟷

STATE

Progress, SR queue, chapter state, streak data, exercise history. All client-side.

⟷

API

Next.js API Routes

Thin proxy to Claude API. Keeps API key server-side. Streaming responses for chat.

⟷

AI

Claude API

Tutor chat, wrong answer explanations, new content generation. Sonnet for speed, Haiku for quick tasks.

CONTENT

Static JSON / MDX files

Chapters, exercises, vocab lists, character data — all stored as structured JSON in the repo. Claude generates new content into the same format, which you can review and add.

Claude Integration — 3 Roles

🤖 Tutor Chat

Always available. You type in German (or English if stuck). Claude responds in German with inline translations for hard words.

Example:

Du: Was bedeutet "Versammlungsfreiheit"?

Claude: Das ist das Recht, sich friedlich zu versammeln (to gather peacefully). Es steht in Artikel 8 des Grundgesetzes...

Model: Sonnet (fast, conversational)

💡 Wrong Answer Explainer

When you get an exercise wrong, a "Warum?" button appears. Claude explains — in simple German — why the right answer is right and your answer was wrong.

Example:

Deine Antwort: Der Bundeskanzler wählt die Minister.

Richtig: Der Bundeskanzler schlägt die Minister vor (proposes). Der Bundespräsident ernennt sie (appoints them).

Model: Haiku (quick, focused)

🔄 Content Generator

After finishing all 10 chapters: Claude generates new exercises targeting your weak areas, in the same JSON format. New scenarios with familiar characters.

How it works:

1. Analyzes your error history

2. Identifies weak LiD topics + grammar

3. Generates exercises in character voice

4. Adds to your exercise pool

Model: Sonnet (creative, structured output)

Spaced Repetition — SM-2 Algorithm

How it works:

Every exercise answer creates a review card:

  • Got it right → card interval increases (1d → 3d → 7d → 14d...)
  • Got it wrong → card resets to short interval, tagged as "schwach"
  • Each card has an ease factor that adjusts based on your performance
  • Daily practice pulls cards that are due today, prioritizing weak ones

What gets tracked per card:

{
  "question": "Was ist Meldepflicht?",
  "chapter": 1,
  "type": "mcq",
  "tags": ["lid", "bürgeramt"],
  "interval": 3,
  "ease": 2.5,
  "nextReview": "2026-03-25",
  "streak": 2
}

Key Technical Decisions

Content as JSON

Chapters, exercises, vocab — all in typed JSON files in the repo. Easy to version, review, and extend. Claude generates in the same schema.

Zustand for state

Lightweight state management. Persists to localStorage via middleware. No Redux overhead.

Framer Motion

For exercise animations — drag-and-drop sentence building, card flips, progress celebrations. Makes it feel like a game.

Anthropic SDK

@anthropic-ai/sdk in API routes. Streaming for tutor chat, structured JSON output for content generation.
